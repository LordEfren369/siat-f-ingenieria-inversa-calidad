# Ingeniería Inversa

Esta carpeta contiene la documentación generada a partir del análisis del sistema existente SIAT-F.

## Objetivo

Registrar el proceso de ingeniería inversa aplicado al sistema, identificando módulos, clases, relaciones, vistas y componentes relevantes.

## Actividades principales

- Revisión de documentación existente.
- Análisis de diagramas UML.
- Identificación de módulos funcionales.
- Identificación de clases principales.
- Análisis de relaciones entre clases.
- Documentación de hallazgos.
- Propuesta de mejoras para el aseguramiento de calidad.

## Resultados esperados

- Descripción de módulos.
- Lista de clases principales.
- Relaciones entre componentes.
- Diagramas UML actualizados.
- Evidencias del análisis realizado.
